## PHASE 1 — Run the Notebook & Get Your Files

### Step 1 — Open your notebook on Google Colab

1. Go to [colab.research.google.com](https://colab.research.google.com)
2. Click **File → Upload notebook**
3. Upload `Intelligent_Credit_Risk_Scoring.ipynb`

### Step 2 — Run the full notebook (top to bottom)

- Click **Runtime → Run all**
- When it asks for your Kaggle API key (opendatasets), enter your Kaggle username
  and API key. Get it from: [kaggle.com/settings](https://www.kaggle.com/settings) → API → Create New Token → download `kaggle.json`
- The notebook will run for approximately **45–90 minutes** (Optuna takes the longest)
- You will see outputs for every section

### Step 3 — Download your generated files from Colab

After the notebook finishes, run this in a **new cell at the bottom**:

```python
# Run this in Colab to zip and download everything
import shutil
shutil.make_archive('model_artifacts', 'zip', 'model_artifacts')

from google.colab import files
files.download('model_artifacts.zip')
files.download('submission.csv')
```

You now have:
- `model_artifacts.zip` — all your trained model files
- `submission.csv` — your Kaggle submission file

### Step 4 — Put the files in the right place

1. Unzip `credit-risk-scoring.zip` (what you downloaded from me)
2. Unzip `model_artifacts.zip` into the `model_artifacts/` folder inside it
3. Copy `submission.csv` into the root of `credit-risk-scoring/`

## PHASE 2 — Test Flask App Locally

Before deploying online, verify it works on your own machine.

### Step 5 — Install Python (if not already installed)

Download Python 3.10+ from [python.org/downloads](https://python.org/downloads)

Verify in terminal/command prompt:
```bash
python --version
# Should print: Python 3.10.x or higher
```

### Step 6 — Open terminal and go to the project folder

**Windows:**
```
Win + R → type "cmd" → press Enter
cd C:\Users\YourName\Downloads\credit-risk-scoring
```

**Mac/Linux:**
```bash
cd ~/Downloads/credit-risk-scoring
```

### Step 7 — Create a virtual environment

```bash
python -m venv venv
```

Then activate it:

**Windows:**
```bash
venv\Scripts\activate
```

**Mac/Linux:**
```bash
source venv/bin/activate
```

You will see `(venv)` appear at the start of your terminal line. This means it worked.

### Step 8 — Install all dependencies

```bash
pip install -r requirements.txt
```

This will take 3–5 minutes. Wait for it to finish.

### Step 9 — Run the Flask app

```bash
cd flask_app
python app.py
```

You should see:
```
[CreditRiskScorer] Loaded — 5× LGB + 5× XGB + meta-learner
[app] CreditRiskScorer ready.
 * Running on http://0.0.0.0:5000
```

### Step 10 — Open the app in your browser

Go to: **http://localhost:5000**

You should see the dark-themed credit risk scoring dashboard.

**Test it:**
- Enter `500000` in Credit Amount
- Enter `150000` in Annual Income
- Enter `0.65` in External Score 2
- Click **Score Applicant**
- You should get a credit score, risk band, and decision

**Test the API directly:**

Open a new terminal window and run:
```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d "{\"AMT_CREDIT\": 500000, \"AMT_INCOME_TOTAL\": 150000, \"EXT_SOURCE_2\": 0.65}"
```

You should get a JSON response with `default_probability`, `credit_score`, `decision`, etc.

Stop the server with `Ctrl + C` when done.

## PHASE 3 — Push to GitHub

### Step 11 — Create a GitHub account (if you don't have one)

Go to [github.com](https://github.com) and sign up for free.

### Step 12 — Install Git

Download from [git-scm.com/downloads](https://git-scm.com/downloads)

Verify:
```bash
git --version
```

### Step 13 — Install Git LFS (for large model files)

The `.pkl` and `.txt` model files are large (some > 50 MB). GitHub blocks files over 100 MB.
Git LFS handles this.

```bash
git lfs install
```

Download Git LFS from [git-lfs.com](https://git-lfs.com) if the command is not found.

### Step 14 — Create a new repository on GitHub

1. Go to [github.com/new](https://github.com/new)
2. Repository name: `credit-risk-scoring`
3. Description: `Enterprise ML Pipeline for loan default prediction. LightGBM + XGBoost + Stacking Ensemble with Flask REST API.`
4. Set to **Public**
5. Do NOT check "Add README" (you already have one)
6. Click **Create repository**

### Step 15 — Set up Git in your project folder

Go back to your terminal (in the `credit-risk-scoring/` folder, NOT inside `flask_app/`):

```bash
cd ..   # if you're still inside flask_app/, go up one level

git init
git lfs track "model_artifacts/*.pkl"
git lfs track "model_artifacts/*.txt"
git add .gitattributes
git add .
git commit -m "feat: Intelligent Credit Risk Scoring System — Enterprise ML Pipeline"
```

### Step 16 — Connect to GitHub and push

Replace `YOUR_USERNAME` with your actual GitHub username:

```bash
git remote add origin https://github.com/YOUR_USERNAME/credit-risk-scoring.git
git branch -M main
git push -u origin main
```

It will ask for your GitHub username and password.

> **Important:** GitHub no longer accepts passwords. Use a **Personal Access Token** instead.
> Go to: [github.com/settings/tokens](https://github.com/settings/tokens) → Generate new token (classic) →
> Check `repo` scope → Copy the token → Use it as your password.

Wait for the upload to finish. Large model files may take 5–10 minutes.

### Step 17 — Verify on GitHub

Go to `https://github.com/YOUR_USERNAME/credit-risk-scoring`

Check that you can see:
- `Intelligent_Credit_Risk_Scoring.ipynb` ✅
- `submission.csv` ✅
- `flask_app/` folder ✅
- `model_artifacts/` folder ✅
- `README.md` renders nicely with badges ✅

## PHASE 4 — Deploy on Render (Free Hosting)

### Step 18 — Create a Render account

Go to [render.com](https://render.com) and sign up with your GitHub account.

### Step 19 — Create a new Web Service

1. Click **New +** → **Web Service**
2. Click **Connect a repository**
3. Find and select `credit-risk-scoring`
4. Click **Connect**

### Step 20 — Configure the deployment

Fill in these settings:

| Setting | Value |
|---------|-------|
| **Name** | `credit-risk-scoring` |
| **Region** | Choose nearest to you |
| **Branch** | `main` |
| **Runtime** | `Python 3` |
| **Build Command** | `pip install -r requirements.txt` |
| **Start Command** | `gunicorn --chdir flask_app app:app --workers 2` |

### Step 21 — Add environment variable

Scroll down to **Environment Variables** section:

| Key | Value |
|-----|-------|
| `ARTIFACT_DIR` | `../model_artifacts` |

### Step 22 — Deploy

Click **Create Web Service**.

Render will:
1. Clone your GitHub repo
2. Install all requirements
3. Start the Gunicorn server

Watch the logs. After 2–5 minutes you will see:
```
[CreditRiskScorer] Loaded — 5× LGB + 5× XGB + meta-learner
Listening on 0.0.0.0:10000
```

### Step 23 — Get your live URL

At the top of your Render dashboard you will see a URL like:
```
https://credit-risk-scoring.onrender.com
```

Click it — your app is now live on the internet!

## Troubleshooting

**"ModuleNotFoundError: No module named 'lightgbm'"**
→ Make sure your virtual environment is activated and you ran `pip install -r requirements.txt`

**"FileNotFoundError: model_artifacts/..."**
→ The `model_artifacts/` folder is empty. Run the notebook first and copy the files in.

**Render build fails with memory error**
→ The free Render tier has 512 MB RAM. Your models combined may exceed this.
Solution: Upgrade to the $7/month Starter plan, or load only 2–3 fold models instead of 5.

**Flask shows "Model not loaded"**
→ Check that `ARTIFACT_DIR` environment variable is set correctly in Render dashboard.

**Git push rejected (file too large)**
→ Make sure you ran `git lfs install` and tracked the large files before committing.
